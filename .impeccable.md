## Design Context

### Users
Photography enthusiasts, potential clients, peers in the street and documentary photography community. They browse intentionally — not scrolling for entertainment but looking at work with attention. Primary context: desktop, evening/night, unhurried. Mobile is secondary but must not feel amputated. They've seen a hundred photography portfolio sites and are sensitive to when a design is trying too hard.

### Brand Personality
Three words: **measured, documentary, unsparing.**

Filat Astakhov is a street photographer — the work is direct, urban, unsentimental. The interface must match. It should feel like the interior of a serious photography gallery or a well-made photobook (think Aperture Foundation, Magnum editorial output) — not a commercial portfolio, not a social media feed, not a wedding photography site. The shell nearly disappears. The photograph is the content.

### Aesthetic Direction
**Dark-first fine art brutalism.**

- Dark mode is the primary experience. The darkness is not "cool developer aesthetic" — it is what makes photographs breathe. A white-background gallery forces the eye to adjust; dark lets images speak at their own luminance. Light mode available via toggle for accessibility and preference.
- Sharp rectangles everywhere. No border-radius. Ever. Borders define structure — 1px on dark surfaces, 2px for emphasis.
- Red (`#cc0000` and variants) is surgical. It marks one active state, one accent, nothing more. It earns its appearance.
- Three-color discipline enforced strictly: near-black, near-white, red. No grays introduced casually — only as precise steps within the palette.
- Layout is architectural: grid columns, deliberate asymmetry in hero areas, the kind of structure that makes an image feel placed rather than dropped.

**Anti-references (explicitly avoid):**
- Tumblr-blog softness: pastel overlays, flowing typography, decorative whitespace
- Wedding photography sites: scripts, rounded corners, soft shadows, blush tones, cards with heart icons
- Generic dark developer portfolios: neon accents, glassmorphism, blue-purple gradients
- Busy grids where every photo competes at equal weight

**References that capture the right feeling:**
- [lorenzobocchi.com](https://shotoniphone.lorenzobocchi.com/) — restraint, photography as the only content, multiple view modes
- [detroit.paris](https://www.detroit.paris/) — bold uppercase architecture, luxury restraint, portfolio-first confidence
- [siemprericc.com](https://siemprericc.com/) — dark/light toggle, all-caps navigation, uncluttered hierarchy

### Typography

**Font stack (do not substitute without explicit discussion):**
- **Display / headings**: Barlow Condensed (Black, ExtraBold) — condensed grotesque with press-photography energy; reads like a contact sheet caption or Magnum photo credit
- **Body / UI**: Figtree — clean, unobtrusive, built for readability; doesn't compete with the photography
- **Monospace / EXIF / metadata**: Azeret Mono — distinctive personality for technical data; avoids the Space Mono cliché

These fonts were specifically chosen to avoid the Space Grotesk + Space Mono defaults used in the prototype mockups.

**Scale principles:**
- Use `clamp()` for display/hero headings on content pages
- Fixed `rem` scale for UI elements (nav, labels, admin panels)
- EXIF and metadata: small, monospace, restrained — data annotation, not feature
- Minimal type size steps, high contrast between levels

### Color System (OKLCH)

Dark mode surfaces:
- `--bg`: `oklch(0.10 0.006 25)` — near-black, faintly warm
- `--surface`: `oklch(0.14 0.006 25)` — slightly lifted surface for panels
- `--text-primary`: `oklch(0.95 0.006 25)` — near-white, warm
- `--text-secondary`: `oklch(0.60 0.006 25)` — muted, same hue family
- `--border`: `oklch(0.22 0.006 25)` — visible but quiet
- `--red`: `oklch(0.48 0.20 25)` — the accent; use sparingly
- `--red-light`: `oklch(0.62 0.22 25)`
- `--red-dark`: `oklch(0.32 0.18 25)`

Light mode surfaces:
- `--bg`: `oklch(0.98 0.006 25)` — near-white, warm
- `--surface`: `oklch(0.94 0.006 25)`
- `--text-primary`: `oklch(0.10 0.006 25)`
- `--text-secondary`: `oklch(0.45 0.006 25)`
- `--border`: `oklch(0.82 0.006 25)`
- Red values identical across modes

### Design Principles

1. **The frame disappears.** Every interface decision asks: does this help the photograph, or does it compete? If it competes, remove it.
2. **Structure through borders, not decoration.** No shadows, no cards, no rounded containers. Lines define space.
3. **Red earns its appearance.** One active nav state, one hover trigger, nothing ambient. If red appears more than twice on a screen, something is wrong.
4. **Dark by default, light on demand.** Theme toggle is always accessible. Both themes use identical structure — only surface luminance changes.
5. **Data is annotation, not feature.** EXIF metadata and tags exist to contextualize the image, not to show off technical richness. Present them quietly.
